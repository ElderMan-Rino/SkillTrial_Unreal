## 💡 핵심 기술 및 아키텍처 적용 (Focus on Architecture)

프로젝트에 적용된 핵심적인 디자인 패턴 및 설계 원칙을 강조했습니다.

### 1. 컴포넌트 기반 아키텍처 리팩토링 (Component Pattern)

- **`Character`**와 **`Enemy`** 액터의 핵심 기능을 독립적인 **`UActorComponent`**로 분리하여 **컴포넌트 패턴(Component Pattern)** 기반으로 리팩토링했습니다.
- **목표:** 클래스 상속의 경직성을 해소하고, 기능별 **응집도(Cohesion)**를 높여 다른 액터 클래스에서 기능 **재사용성**을 확보했습니다.
- **컴포넌트:** 이동 로직, 전투 로직, AI 상태 처리 등이 개별 컴포넌트로 구현되었습니다.

### 2. 느슨한 결합을 위한 메시지 기반 통신 (Observer Pattern)
- **옵저버 패턴(Observer Pattern)**의 구현체인 **`Delegate`**를 사용하여 이벤트 기반 통신을 설계했습니다.
- **적용:** 캐릭터의 상태 변화 (예: 피격, 사망, 아이템 획득) 또는 AI 상태 전이 시, 해당 컴포넌트가 **Delegate를 발행(Broadcast)**하고, 필요한 다른 컴포넌트들이 이를 **구독(Bind)**하여 상태를 처리합니다.
